2. 事务
```text
与关系型数据库不一样,保证的并非是统一成功/统一失败.
redis 事务保证的是
1. 所有命令都会放到一个缓存队列中缓存起来
2. 接收到 exec 命令以后事务开始执行, 并且不会因为某个命令失败而导致其他命令无法执行
3. Redis 事务只保证：命令“按顺序、一次性执行”，不保证原子成功，也不支持回滚。
```
